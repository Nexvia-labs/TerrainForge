# Contributing to TerrainForge

> Thank you for your interest in contributing to TerrainForge! 
> 
> TerrainForge is an offline-first procedural terrain generation engine designed for accessibility, deterministic world generation, and smooth performance on low-end devices.

Every contribution—whether code, documentation, bug reports, or feature ideas—is highly appreciated.

---

## 📌 Project Principles

| Principle | Description |
| :--- | :--- |
| **Offline First** | No internet required for core functionality |
| **Deterministic** | Same Seed = Same World |
| **Lightweight** | Optimized for low-end devices |
| **Educational** | Easy to understand and modify |
| **Open Source** | Community-driven development |

## 💡 Feature Requests

Before requesting a feature, please answer these questions in your issue:

* **What problem does it solve?**
* **Why should TerrainForge include it?**
* **Is there another way to solve it?**
* **Does it affect offline functionality?**

## 💻 Coding Guidelines

### Branch Naming

| Purpose | Example |
| :--- | :--- |
| **Feature** | `feature/obj-export` |
| **Fix** | `fix/water-memory` |
| **Documentation** | `docs/update-readme` |
| **Optimization** | `optimization/render-speed` |

### Commit Messages

**✅ Good**
* `Improve terrain rendering performance`
* `Add OBJ exporter`
* `Fix seed synchronization`
* `Reduce memory allocation`

**❌ Avoid commits like:**
* `update`
* `fixes`
* `changes`
* `final version`
* `new commit`

---

## 📦 Pull Request Checklist

Before submitting a Pull Request, verify:

- [ ] Code builds correctly
- [ ] Existing features still work
- [ ] README updated (if needed)
- [ ] No unnecessary files included
- [ ] Clear commit history
